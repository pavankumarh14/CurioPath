EXAMPLE WITH A GAP — "Kubernetes" topic
======================================

If the user submits a topic NOT in the mock pool (e.g. "Kubernetes"),
the Source-Finder falls back to HN search. For some sub-topics it may
find nothing relevant — the Sequencer should flag these as gaps.

======================================
PREREQUISITE ORDER — Why this matters
======================================

Without topological sort (wrong order):
  1. State Management       ← WRONG: needs Hooks first
  2. Hooks and Side Effects
  3. State and Event Handling
  4. JSX and Component Basics
  5. React Router
  6. JavaScript ES6+        ← WRONG: must be first

With topological sort (correct order):
  1. JavaScript ES6+        ← no prerequisites
  2. JSX and Component Basics  ← needs JS only
  3. State and Event Handling  ← needs JSX
  4. React Router           ← needs JSX (independent of State)
  5. Hooks and Side Effects ← needs State
  6. State Management       ← needs Hooks

A learner following the wrong order will hit concepts they haven't
learned yet. The Sequencer prevents this.

SQLITE VERIFICATION COMMANDS
======================================

After submitting "Learn React", verify each agent worked:

  # Check Scoper saved sub-topics
  sqlite3 backend/data/learnswarm.db "SELECT name, prerequisites, estimated_hours FROM sub_topics"

  # Check Source-Finder saved sources
  sqlite3 backend/data/learnswarm.db "SELECT s.title, s.type, st.name FROM sources s JOIN sub_topics st ON s.sub_topic_id = st.id"

  # Check Quality-Rater scores
  sqlite3 backend/data/learnswarm.db "SELECT s.title, rs.overall_score, rs.keep FROM rated_sources rs JOIN sources s ON rs.source_id = s.id ORDER BY rs.overall_score DESC"

  # Check Sequencer built the path
  sqlite3 backend/data/learnswarm.db "SELECT total_estimated_hours, fits_timeline, gaps FROM learning_paths"

  # Check all findings
  sqlite3 backend/data/learnswarm.db "SELECT capability, confidence, summary FROM findings ORDER BY created_at ASC"

Expected flow: 4 findings in order — scoper (confidence ~0.88), source-finder, quality-rater, sequencer (model='algorithm')
